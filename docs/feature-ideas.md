## Группа A — Уже заложено в коде, нужно только доделать

### A1. Обнаружение конфликтов (validateConflict)
**Заглушка:** `src/domain/services/item-service.ts:199`  
Функция вызывается при каждом создании/обновлении, но тело пустое.  
**Что сделать:**
- При создании/переносе задачи проверять пересечение по времени с другими events
- Показывать предупреждение в модалке (не блокировать, но предупреждать)
- Опционально: выделять конфликты в calendar-виде цветом

---

### A2. Полная поддержка повторяющихся событий
**Стаб:** `handleRecurrenceStub()` в item-service.ts  
**Что уже есть:** FREQ/INTERVAL/BYDAY/COUNT парсинг, expansion  
**Чего не хватает:**
- `UNTIL` — повторять до даты (сейчас только COUNT)
- Редактирование одного экземпляра серии с сохранением исключения (`editScope: "this"` уже в типах, но UI не отдаёт выбор пользователю явно)
- Удаление одного экземпляра из серии (сейчас только "удалить всё")
- Диалог выбора: «Изменить это событие / Это и все следующие / Всю серию»

---

### A3. Авто-синхронизация подписок на календари
**Есть:** ручной sync через `POST /api/subscriptions/[id]/sync`  
**Нет:** автоматического запуска  
**Что сделать:**
- Хранить интервал синхронизации (каждые N минут)
- В Electron: запускать через `setInterval` в main.cjs
- Показывать `errorMsg` в UI (поле есть в схеме, но нигде не отображается)
- Индикатор «последняя синхронизация: X минут назад»

---

### A4. Родительские задачи / Подзадачи
**Поле `parentId`** уже в схеме Prisma, но нигде не используется.  
**Что сделать:**
- Отображение вложенных задач в модалке и agenda-виде
- Создание подзадачи из контекстного меню или кнопки «+» внутри задачи
- Прогресс-бар на родительской задаче (X/N подзадач выполнено)
- Раскрыть/свернуть в списке

## Группа C — UX-улучшения

### C1. Drag-and-drop перенос событий
react-big-calendar поддерживает DnD через addon `react-big-calendar/lib/addons/dragAndDrop`.  
Перетащить событие на другой день/время — базовое ожидание пользователя.

---

### C2. Глобальный поиск
- Shortcut `Cmd/Ctrl+K` — поиск overlay
- По названию, описанию, тегам, проекту
- Переход к дате события при выборе результата
- Дополнительно: поиск по ссылкам

---

### C3. «Сегодня» / Focus View
Минималистичный экран:
- Только задачи и события на сегодня
- Быстрое переключение статусов
- Прогресс дня (X/N задач)
- Хорошо для начала рабочего дня

---

### C4. Приоритеты задач
Добавить поле `priority: LOW | NORMAL | HIGH | URGENT` в Item.  
- Цветовая/иконочная индикация в calendar-виде
- Сортировка в agenda по приоритету
- Фильтр по приоритету

---

### C5. Bulk-операции
- Выбор нескольких элементов в agenda-виде
- Массовое: удалить, сменить статус, перенести в проект, тегировать

---

### C6. Таймер / Pomodoro
- Кнопка «Начать работу» на задаче
- 25/5 таймер (настраивается)
- После завершения: предложить отметить задачу выполненной
- История сессий в Dashboard

---

### C7. Хлебные крошки / контекст в модалке
- В модалке задачи показывать путь: Проект → Серия → Задача
- Быстрый переход к проекту

## Группа D — Desktop / Electron

### D1. Системные уведомления
- Electron `Notification` API
- Напоминание за N минут до события
- Настройка в settings: за сколько минут, звук вкл/выкл

---

### D2. System Tray
- Иконка в трее с быстрым доступом
- Меню: «Задачи на сегодня», «Добавить задачу», «Открыть»
- Показывать количество задач на сегодня на иконке

---

### D3. Backup / Restore
- «Экспортировать данные» → JSON или SQLite-файл
- «Восстановить из файла»
- Автобэкап при каждом запуске в папку `%APPDATA%/Dinox/backups/`

---

### D4. Auto-launch при старте системы
- Опция в settings: «Запускать при входе в Windows»
- `app.setLoginItemSettings()` в Electron

---

## Группа E — Интеграции

### E1. CalDAV двусторонняя синхронизация
Поля `externalSource`/`externalId` уже есть в схеме для Project, Tag, Item.  
- Sync с Google Calendar, Apple Calendar, Nextcloud
- Разрешение конфликтов (last-write-wins или merge)

---

### E2. Webhooks / API-ключи
- Внешние системы могут создавать/обновлять задачи через REST
- Настройка в settings: выдать API-ключ
- Полезно для интеграции с Zapier, n8n, Make

---

### E3. Export в разные форматы
- **CSV**: для анализа в Excel/Sheets
- **PDF**: отчёт по проекту или неделе
- **Markdown**: задачи недели в формате чеклиста

## Не стоит сейчас

- **Мультипользовательский режим** — архитектура поддерживает, но требует auth, backend, хранилище → отдельный продукт
- **Mobile app** — пока база веб-приложения не устаканилась
- **Кастомные поля на задачах** — излишняя сложность для текущего масштаба
